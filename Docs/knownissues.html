# Known issues and limitations

The Z Open Editor for COBOL and PL/I extension that has some known issues and limitations. This page provides a reference to the status of the current release. If you want to provide feedback or report more issue, please use our public github repository to do so. Any report and feedback would be very much appreciated.

# Java Runtime required

The language servers provided with this extension require a valid Java runtime to start correctly. If the right version of Java cannot be found in the current user's path or no JAVA_HOME environment variable has been defined you might see error messages in the Output view of VS Code. There can also be other reason for such errors such as Java was not able to find enough free memory on your system.

Go through all the steps described in the Getting Started page for configuring Java to try out different opetions.

# JCL

There is currently no language server for JCL: When you open JCL files, syntax highlighting is enabled, but no additional Language Server capabilities are available for these files yet. The observable limitations include:

  • No contents are shown in the Outline view,
  • Ctrl+Space shows code templates, but no syntactical completions
  • Many menu options are unavailable.

# COBOL Language Server Status

The following list shows the general status of our language server implementation. Use these lists to check that the feature that you are looking for has been implemented or if there are known issues. Let is know if you do experience any additional issues.

# COBOL Program files working features

  • syntax highlighting
  • syntax errors
  • document highlighting
  • hover
  • go to declaration
  • find all references in file as well as across the multiple files
  • peek references in file as well as across the multiple files
  • rename symbol in file as well as across the multiple files
  • outline view
  • code completion
  • snippets
  • copybook resolving and copybook hover support
  • document link support to open copybook using ctrlcmd+click
  • embedded languages support such as SQL, CICS, DLI
  • cobol program metrics
    • status bar metrics data
    • cobol metrics table view
    • cobol metrics threshold
    • warnings and violations
    • live changes of metrics data

# COBOL Program files limitations

  • None currently reported

# COBOL Copybook files working features

  • find all references in file
  • peek references in file
  • rename symbol in file
  • outline view
  • syntax highlighting
  • code completion
  • snippets
  • embedded languages support such as SQL, CICS, DLI

# COBOL Copybook files limitations

  • no support for syntax errors
  • no support for hover
  • no support for go to declaration
  • no support for find all references across the multiple files
  • no support for peek references across the multiple files
  • no support for rename symbol across the multiple files
  • no support for cobol program metrics
    • no support for status bar metrics data
    • no support for cobol metrics table view

# PL/I Language Server Status

# PL/I Program files working features

  • syntax highlighting
  • syntax errors
  • document highlighting
  • hover
  • go to declaration
  • find all references in file as well as across the multiple files
  • peek references in file as well as across the multiple files
  • rename symbol in file as well as across the multiple files
  • outline view
  • code completion
  • include file resolving and include file hover support
  • document link support to open include file using ctrlcmd+click

# PL/I Include files working features

  • syntax errors (lexing and parsing)
  • find all references in file
  • peek references in file
  • rename symbol in file
  • outline view
  • syntax highlighting
  • code completion
  • embedded languages support such as SQL, CICS, DLI

# PL/I Include files limitations

  • no support for showing syntax errors for resolving variables
  • no support for hover
  • no support for go to declaration
  • no support for find all references across the multiple files
  • no support for peek references across the multiple files
  • no support for rename symbol across the multiple files
